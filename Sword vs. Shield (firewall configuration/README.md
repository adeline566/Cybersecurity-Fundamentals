                                    Sword vs. Shield


# Overview

This assignment focused on two fundamental cybersecurity concepts: network reconnaissance (Sword) and network defense (Shield). In the first part, I used Nmap to enumerate hosts, identify operating systems, discover open ports, and determine running services within a lab network. In the second part, I configured pfSense firewall rules to restrict network communication and observed how those controls affected network visibility and accessibility. Additionally, I used Wireshark to analyze network traffic generated during scanning activities and explored vulnerability assessment using Nessus.

### Step 2: Traffic Analysis with Wireshark

While Nmap was actively scanning the network from External Kali, Wireshark was executed on Internal Kali to monitor and analyze network traffic generated during reconnaissance.

#### Observations

A total of 5,371 packets were captured during the scan:

* TCP	4,572
* ARP	743
* ICMP	39
* UDP	22
* FTP	1

The majority of captured traffic consisted of TCP packets generated by Nmap's SYN scan process. Numerous TCP SYN packets were sent to target systems including Ubuntu, pfSense, and Windows Server 2022. Open ports responded with SYN-ACK packets, while closed ports responded with RST-ACK packets. This behavior allowed Nmap to accurately determine port states without completing the full TCP handshake.

ICMP traffic was also observed as part of host discovery activities. Echo Request packets were transmitted to determine whether hosts were online, and active systems responded with Echo Reply packets. One destination returned an unreachable response because Internal Kali and External Kali resided on different network segments.

ARP traffic was heavily utilized throughout the scan. Multiple ARP requests were broadcast to resolve IP addresses into MAC addresses before communication could occur. For example, the host at IP address 192.168.10.18 responded with the MAC address 00:15:5d:40:57:32. These exchanges demonstrated the critical role ARP plays in local network communication.

Overall, the packet capture provided valuable insight into how reconnaissance tools interact with network devices and how scanning activity can be detected and monitored through traffic analysis.

# Lessons Learned

This assignment strengthened my understanding of both offensive and defensive cybersecurity techniques. I learned how Nmap can be used to gather detailed information about hosts, operating systems, open ports, and running services during network reconnaissance. By analyzing packet captures in Wireshark, I gained a deeper understanding of how TCP, ICMP, and ARP traffic support host discovery and service enumeration. I also observed how scanning activity generates recognizable traffic patterns that can be detected and monitored by defenders.

Through the firewall configuration exercises, I learned how network security controls can significantly limit an attacker's visibility and access to internal resources. Blocking ICMP traffic reduced host discovery capabilities, while restrictive access control rules prevented unauthorized communication across the network. Finally, using Nessus demonstrated the importance of vulnerability assessments in identifying potential security weaknesses before they can be exploited. Overall, this assignment highlighted the constant balance between attack techniques and defensive controls and reinforced the importance of layered security in protecting network environments.
